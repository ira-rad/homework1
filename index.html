Основні команди Git

  • git config --global user.name ... - вказати ім'я користувача.
  • git config --global user.email ... - вказати пошту користувача.
  • git init - створити локальний репозиторій.
  • git status - показати стан репозиторію, переглянути зміни.
  • git add - додати всі файли, в яких були зміни, а також нові файли.
  • git commit - закомітити зміни у файлах.
  • git log - перегяд історії комітів.
  • git checkout -дозволяє переходити по гілках та комітах.
  • git branch - це свого роду менеджер гілок. За допомогою цієї команди можна перераховувати гілки, створювати нові, видаляти та переіменовувати їх.
  • git push - використовуються для завантаження змін з локального репозиторію у віддалений.
  • git pull - використовується для завантаження змін з віддаленого репозиторію в локальний.
  • git merge - використовують для злиття гілок в одну.
  • git clone - використовують для отримання копії існуючого Git-репозиторію.
  • .gitignore файл потрібен для того, щоб приховати папки та файли від системи контролю версій Git. Зазвичай приховують папки конфігурації, а також тимчасові папки та фaйли.
  • git revert - застосовується для для скасування непотрібного коміту.
  • git reset - застосовується для скасування змін.
  • git rebase - використовують для перезапису історії комітів при злитті гілок.
  • git cherry-pick - застосовують щоб переносити потрібні коміти між гілками.
  • git stash - використовують для тимчасового зберігання всіх незакомічених змін для очищення робочої директорії без потреби комітити незакінчену роботу в нову гілку.
